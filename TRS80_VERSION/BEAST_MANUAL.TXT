**************************
BEAST for the TRS-80 M1/M3
**************************

By Steve Lewis (Xiphod), April 2026
contact.steve.usa@gmail.com

Intended to be loaded and executed at address 0x5200.

I first implemented this on the POLY-88, which uses a "VTI"
display adapter that has Greek symbols character set.
Since it had the same 2x3 semigraphics as the TRS-80, I
figured it would be an easy port.

The main challenge was the POLY-88 has a 60Hz timer
interrupt (built into the VTI).  I was using that for
RNG elements.  The stock TRS-80 Model 1 doesn't.

I know the Expansion Unit adds one, and the Model 3
have a clock timer.  Anyway, I decided to go Apple ][
style and just count cycles as my timer, and do the
"user presses key at startup" to init the seed.

Because there are some symbol differences between the M1 and
M3, I decided to try to "auto-detect" the M3 by looking for
its timer (but I don't actually use the timer after that).
If I detect this timer counter, then I'll load in the M3
symbol set.   If this detection is wrong (maybe your timer
chip is broken), you can still press ENTER at the starting 
screen to cycle between M1/M3.

Aside from a few symbol changes, the game otherwise
plays like the POLY-88 version!

BACKGROUND
==========
BEAST is a scaled-down clone of a game by the same name 
available on the MS-DOS platform from 1984.  It is a 
single-scene text mode game.  The MS-DOS version used 80-column
mode but made clever use of "two character wide icons" to
effectively make it a 40-column game.  For this TRS-80 
version, a combination of 2x3 pixel semigraphic symbols and 
regular text mode is used.


QUICK OBJECTIVE / HOW TO WIN
============================
Eliminate all BEAST to advance to the next round.
Your basic objective is to push the blocks around, trap
a beast in between two blocks, and then SMASH them!

Eggs and infants may evolve into new BEAST if 
left unchecked; they are on a timer!

Avoid touching EGGS or INFANTS (they are poisonous). 
But you can use blocks on them!  Just don't let the 
BEAST catch you!

You can also PULL blocks to maneuver them in creative ways.
You cannot move the TREES.

Press BREAK if you get really stuck (that resets the game).


STARTING THE GAME
=================
TL;DR:  Press any key to begin (after optional mode selections).


The startup of this TRS-80 version is more sophisticated 
than it seems!

First, the software tries to detect if you are using a 
Model 3.  It does this by looking for the timer increment 
used on that system.  If detected, you will see an "M3" at 
the top left corner during the start.

If not detected, you will instead see "M1" at the top left 
corner, for Model 1.  During testing, we re-learned that not 
all M1's have the 8-bit upper/lower case Character ROM.   So 
we also try to detect for this.  If detected, you will see 
an "8" otherwise you will see "7" (for 7-bit video mode, where 
the 6-bit is not available).

You can override these detections by pressing M for the M1/M3
mode toggle, and T for the 8-bit/7-bit video support.

In addition, you will see a "snake" encircling the startup
instructions.  Since the software doesn't actually make use of
the timer for RNG, this spiraling snake is your RNG seed 
decider.   (for fun in the emulator, hold F12 to see it spin
fast)

Make your MODE and TOGGLE decisions, or accept the 
auto-detected settings, and press any (OTHER) key to 
start the game!


DETAILED GAME RULES
===================
You (PLAYER) start in an arena, generally spawned onto the 
left side.  Two or more BEAST are spawned on the right side.

Within the arena is randomly placed two primary 
types of objects:

(1) BLOCKS, which you can PUSH or PULL.  To PUSH a block, you
just move into them and push them in the direction you are
going.  You can even push multiple blocks!  However, you
cannot push into TREES or BEAST (with one exception, 
see below), nor can you push into the arena border walls.

(2) TREES, which are strongly rooted and cannot be moved.

Near each BEAST is spawned an EGG.  The BEAST have a strong
sense of smell and will begin seeking you out right away,
to protect their egg.  BEAST can sometimes move diagonally.

NOTE: To simplify the game, I decided that the player can
only move in cardinal directions (4-way) and not diagonal.

Your objective is to squash all BEAST using the BLOCKS.
Keep in mind you can pull blocks, use this to make little
defensive forts or set traps for the BEAST.  However,
be aware that you cannot pull a BLOCK that is near a BEAST!

At the start of the round, the EGGS will have a random
duration until they hatch into an INFANT (typically like
around 20 seconds).  If you are quick, you can simply
move a BLOCK and rollover an EGG.  But your main objective
is only to smash any BEAST - so best to do so before EGGS
become INFANTS, since INFANTS will also eventually grow
to become BEAST (but they won't spawn additional eggs).

TIP: INFANTS grow into BEAST at staggered times, so new 
BEAST arrivals may occur in waves rather than all at once.
These new arrivals give a big roar when they evolve, you
will notice it!


If a BEAST runs into you - or if you run into a beast by
accident - you are defeated instantly.  Also EGGS and 
INFANTS are poisonous, so don't touch them.  You can 
rollover EGGS with BLOCKS, or squish INFANTS or BEAST 
between two BLOCKs (or the outer arena wall).

TREES do count as BLOCKS, but they are very rooted and 
cannot be moved.

Each round, there is a random number of BLOCKs and TREEs
generated.  There is always at least two BEAST.  Each BEAST
should have an egg near it.  BEAST won't step on or 
harm eggs.


OTHER WAYS TO PLAY
==================

- Build your own castle, stay inside and let the 
  beast roam around all day.  Like a contained ecosystem!

- Box the beast into a cage.  Place a tunnel between
  cages, imagine the Beast as little hamsters!



CONTROLS
========

WASD  Move up, left, down, right
(you cannot move into arena border, trees; do not
move into BEAST, EGGS, or INFANTS)

IJKL  Activate PULL, and any BLOCK opposite your
      direction of movement will be pulled.
      (unless there is a BEAST touching the BLOCK)
 
BREAK Restarts the game, back to the MODE and TOGGLE
      menu and the snake-square animation.
      (use this if you picked the wrong selections,
      or if the gameplay gets stuck somehow)

NOTE: On the MS-DOS version, PULL was activated by
holding SPACEBAR and using direction keys.  The TRS-80
does make scanning for multiple keys at the same time
possible, but the POLY-88 does not, and I decided to keep
the gameplay consistent to the POLY-88 version.



TECHNICAL NOTES
===============
There are two types of beast, one is a little more
aggressive and smarter (and more likely to go diagonal).
The sleepy-beast is only a 40% chance of spawning, but
they will promote into the more aggressive diagonal-moving
beast once they are within about 5 cells of the player.

On the M3, I had to disable the cursor blinking.  Because
my timing is cycle based, that interrupt was wrecking
the player movement reaction (making it a little jerky).

Also in "M3 mode" - some minor timing changes are made 
since that system runs slightly faster M1.

Frankly, the "M1 8-bit" is my favorite version of the game,
I'll consider it the "official canon" of how it was intended
for this system.   The symbol set of the M3 and the M1 7-bit
meant I had to make some compromises.

When an INFANT grows to a full BEAST, the BEAST-banner on
the right side of the screen does a reverse-polarity wave
to get your attention that this event has taken place (it
represents a mighty roar of this now full grown addition).

When you SMASH a beast (or EGG or INFANT), a big SMASH word
pops up on the left.  These are 8x48 pixel graphics.  The 
Python script I used to make these (from 2-color BMP to
2x3 VTI semigraphics) is in the CODE folder.

The BEAST-arrival-flash and SMASH-flash are both 
"non-blocking" so you can still take action while they 
are occuring!  This was an important detail, to avoid
disrupting things like double or even triple SMASHes!!

If you defeat all the active BEAST, then a YOU WIN
banner flashes.  This one is blocking, since it represents
the end-game win state.  Savor your victory!!

The ASM code was compiled using Alfred Arnold
AS assembler (asl.exe).

poly88_vti_encode.py is a Python script used to
convert 2-color BMP file into a DB array of hex
values corresponding to suitable 2x3 VTI symbols
that match the BMP.  This made it easy to add the
pixel graphics into the game.


SPECIAL FEATURES
================
On startup, "3...2...1..." countdown is used as a round
starting delay, and to help highlight where your character
will be starting from within the arena.

If you get defeated by a beast or run into a poison item 
(egg or infant), the object defeating you will flash for
a few moments before the next round.  Touching a poison
item (egg or infant) displays SKULL GRAPHIC on the sides.
Touching a beast displays DEATH GRAPHIC on the sides.

The objective on each round is to defeat any active BEAST
by smashing them.  Once you do that, the YOU WIN graphic
shows and goes to the next round.  This graphic flashes
by inverting the VTI symbols briefly, and it first
flashes quickly then gradually a little slower.

EGGS do not need to be defeated nor INFANTS.   However, 
keep in mind that EGGS are easy to wipe, since you can 
just run them over with BLOCKS (i.e. don't have to actually
smash them against a tree or other block).

Anytime you smash something, the word SMASH flashes
on the left.  It's subtle, but this graphic is
"non-blocking" - so if you end up doing a chain of
say three smashes in a row, this graphic won't stutter
or lag the player in doing so!

In the code there is:
BLOCKCHR        EQU     0FFH    ; WHITE BLOCK
I went back and forth on whether the BLOCK symbol should
be 00H or FFH.  If you use 00H, that is a full 2x3 pixel
block that fills the entire cell.  If you use FFH, you
get a little spacing between the blocks.   To me, 00H
feels a little imbalanced.  But FFH loses the "solid wall"
feel.  On the POLY-88, I kept the spacing. But for the
TRS-80, I leaned towards keeping the solid wall.

The POLY-88 has an Omega symbol used for beast.  For the 
TRS-80 version, the symbol is intended as a scorpion symbol.
Due to technical limitations, for the M1 7-bit and 
M3 runtimes, the exact appearance varies.

The POLY-88 also has a PI symbol I had used for the player.
For the TRS-80, I went with lower case o.


MAXBEAST        EQU     8
NUMBEAST        EQU     4
MINBEAST        EQU     2
The code has these three equates, and a lot of time was
spent supporting this.  Because each BEAST spawns also
with an EGG that can potentially become a BEAST, then
MAXBEAST should always be at least double NUMBEAST.
NUMBEAST sets upper limit of how many BEAST will be
spawned per round.  I've tested with this up to 6, and
the game gets very exciting with 12 BEAST running around.
These CPUs are fast enough that the game engine could handle
more beast, but due to the practical size of the arena,
I felt maximum of 8 was enough.  In any case, the game
is prepared to dynamically scale up in difficult, but
I just wanted a more casual build for now.

For each round, there is roughly 18 to 25 BLOCKs
randomly placed.  For TREES, it is about 8 to 15.
Effort is taken so that nothing is placed on top of
anything else (like BEAST don't spawn into walls,
eggs don't spawn into trees, etc.).

EGGS hatch in about 20 seconds, and after that the 
INFANTS grow into BEAST in a little less time 
(about 12 seconds).

The "attract mode" graphic on the right side that
says BEAST: this will alternate styles in between
each round.



HOW TO LOAD
===========
Using trs80gp.exe, do:

trs80gp -m1 -dx     <-- use TRS-80 Model 1 runtime

trs80gp -m3 -dx     <-- use TRS-80 Model 3 runtime
                      (should get auto-detected,
                      you'll see an M3 at top left)

On M3, at the "Cass?" prompt type L for LOW (500 baud).

At BASIC, type:
SYSTEM <ENTER>
then
BEAST <ENTER>

In the emulator: "Insert" the BEAST.CAS file to the 
"Cassette" or play a WAV file generated from that.

In turbo-mode, this is ~8KB file and will take a
few seconds (over a minute on real hardware at real-time).

Once loaded, type
/ <ENTER>
To auto-run the loaded program (that is at 0x5200).

If you are having trouble with the .CAS or .WAV, then
another option is to load the Intel Hex file directly.
That file is in the CODE sub-folder.  In the emulator,
go to File|Load, look for the .HEX file in the CODE 
folder.  This step just takes that raw binary and puts
it into RAM.  To actually run the code, next you go to 
Debug|Z80 Debugger, and set the "PC" (Program Counter)
to "5200" (for 5200 HEX, the address where the code
is set to be loaded at).
